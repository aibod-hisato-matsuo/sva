# 03. 縦串設計

## 見える化縦串
- ID: vs_001
- ゴール: メーカー差異を越えて家庭エネルギー状態を一貫表示する
- トリガー: 定周期収集またはイベント発生
- 必要データ:
- grid_import_power
- grid_export_power
- pv_generation_power
- battery_soc
- battery_charge_discharge_power
- 判断ロジック:
- 取得ソースの正規化
- 欠損時の推定値生成
- 実測/推定の表示区分
- 日次・月次集計
- 実行対象:
- cloud
- user_app
- admin_app
- UI接点:
- ホームダッシュボード
- 時系列グラフ
- 計測値詳細
- フォールバック: 一部欠損時は推定値表示へ切替し、推定フラグを明示する
- 関連ユースケース:
- uc_001

## 節約提案縦串
- ID: vs_002
- ゴール: ユーザー方針に応じた節約・自家消費の推奨を提示する
- トリガー: 料金時間帯変化、発電余剰、ユーザーモード変更
- 必要データ:
- pricing_plan
- pv_generation_power
- battery_soc
- grid_import_power
- user_policy_mode
- 判断ロジック:
- 料金時間帯判定
- 余剰発電活用優先度判定
- 夜間充電の推奨判定
- 通知の必要性判定
- 実行対象:
- cloud
- user_app
- UI接点:
- 提案カード
- 節約レポート
- モード設定画面
- フォールバック: 外部料金API異常時は直近既知の料金情報で代替し、推定扱いにする
- 関連ユースケース:
- uc_002

## 備え優先縦串
- ID: vs_003
- ゴール: 停電リスク時に蓄電池残量確保を優先する
- トリガー: 停電警報、災害通知、ユーザーの備え優先設定
- 必要データ:
- battery_soc
- outage_alert
- user_policy_mode
- battery_control_capability
- 判断ロジック:
- 停電リスク受信
- 備え優先モードへの遷移判定
- 放電抑制・SOC下限制御判定
- 実行対象:
- cloud
- edge_controller
- user_app
- UI接点:
- 災害備え設定
- 緊急通知
- モード状態表示
- フォールバック: クラウド未接続時は端末の事前設定ルールに従い保守的に動作する
- 関連ユースケース:
- uc_003

## 通信断保護縦串
- ID: vs_004
- ゴール: 通信断・機器異常時にも危険側に倒れず監視可能性を維持する
- トリガー: 通信断検知、機器応答異常、クラウド接続喪失
- 必要データ:
- communication_status
- device_health_status
- last_known_policy
- 判断ロジック:
- 異常区分判定
- 監視継続可能範囲の判定
- 安全側縮退ルール適用
- 復旧後の再同期
- 実行対象:
- edge_controller
- cloud
- admin_app
- UI接点:
- 障害通知
- 運用監視画面
- 機器状態画面
- フォールバック: 端末は最後の安全設定を保持し、クラウド復旧後に差分同期する
- 関連ユースケース:
- uc_004
