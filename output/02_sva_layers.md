# 02. SVAレイヤー設計

## L0: Purpose / Policy
- 意図: 家庭エネルギー主権の実装
- 主要要素:
- ユーザー選択可能性
- メーカー非依存
- 安全側縮退
- 提案主体・代行しないAI
- 制約:
- AIは意思決定を代行しない
- 誤解を招く表示を避ける

## L1: Requirement / Schema
- 意図: 共通意味モデルと要求の定義
- 主要要素:
- 計測値
- イベント
- ユーザーモード
- 推定値フラグ
- 機器能力宣言
- 制約:
- 実測値と推定値を区別できること
- メーカー名ではなく能力差で表現すること

## L2: Logic / AI
- 意図: 提案ロジックと制御ルールの定義
- 主要要素:
- 節約提案
- 備え優先提案
- 売電/自家消費バランス
- 異常検知・保護
- 制約:
- 安全制御は端末側へ依存可能な範囲を限定する
- 提案根拠を説明可能にする

## L3: Orchestration
- 意図: Edge/Cloud/Appの責任分割と同期
- 主要要素:
- データ同期
- 通知フロー
- 障害時縮退
- 承認フロー
- 制約:
- 通信断時も最低限の運転継続
- クラウド障害で危険側に倒れない

## L4: Interface / Adapter
- 意図: Wi-SUN/ECHONET Lite/メーカー差異の吸収
- 主要要素:
- 取得チャネル統一
- プロパティマップ
- コマンド変換
- エラー標準化
- 制約:
- アプリ層にメーカー固有差分を漏らさない
- 追加メーカーを後付けしやすくする

## L5: Execution / Physical
- 意図: 実機接続と物理安全実行
- 主要要素:
- HEMS端末
- PV
- 蓄電池
- スマートメーター
- 家庭内通信
- 制約:
- 端末は安全実行主体
- 通信断時の保守動作を持つ
